## <a name="v0.31.0">v0.31.0</a> - 2026-07-12
### Added
- merge: Add 'spice.merge.ready.command' configuration option to customize what is considered ready-to-merge for a CR.
- merge: Add --fail-fast flag to all merge commands to stop scheduling remaining merge queue work after the first branch failure.
- Add '--no-restack' flag to opt out of automatic restacking for the following commands: branch create, branch edit, branch squash, commit amend, commit create, commit fixup, commit pick, commit split. Each command also gets a configuration option to change the default behavior, and opt-into restack with --restack. See [Configuration](https://abhinav.github.io/git-spice/cli/config/) for details.
- bitbucket: Support self-hosted Bitbucket Data Center instances with the 'spice.forge.bitbucket.kind' configuration option.
### Changed
- github: Replace GraphQL client implementation. While we expect no user-facing impact from this, it's a significant change. If you encounter any regressions, [please report them here](https://github.com/abhinav/git-spice/issues).
- repo sync: When a merge branch is deleted but the local and remote heads differ, include more information in the deletion prompt to help the user make an informed decision.
- merge: Rename 'spice.merge.readyTimeout' configuration option to 'spice.merge.ready.timeout'.
- merge: Rename 'spice.merge.mergeTimeout' configuration option to 'spice.merge.timeout'.
- bitbucket: A custom 'spice.forge.bitbucket.url' is now interpreted as a Bitbucket Data Center instance unless 'spice.forge.bitbucket.kind=cloud' is set.
### Removed
- downstack merge, stack merge: Remove --no-branch-check flag.
### Fixed
- forge: 'spice.forge.kind' configuration option now works with remotes that use Git's 'url.*.insteadOf' rewriting.
- github: Label IDs are resolved in batches instead of one API call per label.
- github: Reviewer and assignee IDs are resolved in batches instead of one API call per identity.
- github: Pull Request label updates, review requests, and assignments are batched to reduce the number of API calls.
- bitbucket: New pull request descriptions are now filled from pull request template files on the default branch.
